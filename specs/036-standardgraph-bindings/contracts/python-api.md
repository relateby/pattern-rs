## Implementation Notes

**`from_gram` location**: Implemented in `python/relateby/pattern/__init__.py` (unified package layer), not in `crates/pattern-core/src/python.rs`. This is because `pattern-core` cannot depend on `gram-codec` (circular dependency — gram-codec depends on pattern-core for types). The unified Python package has access to both subpackages and bridges them with a pure-Python classmethod that calls gram's `parse_gram()` then passes results to `StandardGraph.from_patterns()`.

## Python-Specific Behaviors

- `SubjectBuilder.property()` accepts native Python types (`str`, `int`, `float`, `bool`) directly
- `__repr__` returns `StandardGraph(nodes=N, relationships=N, walks=N, annotations=N)`
- `__len__` returns total element count (nodes + relationships + walks + annotations)
- All `add_*` methods return `self` for chaining (PyO3 `PyRefMut<Self>`)
